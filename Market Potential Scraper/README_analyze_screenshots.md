# DataEye Screenshots 数据分析工具

## 功能概述

这个工具用于分析 `screenshots` 目录下的 DataEye API JSON 数据，并生成包含多个维度的 Excel 分析报告。

## 使用方法

### 1. 基本用法

```bash
python analyze_screenshots.py
```

脚本会自动：
- 扫描 `screenshots` 目录下的所有 JSON 文件
- 按剧名分组（2年数据 + 30天数据）
- 提取关键维度数据
- 生成 Excel 报告（文件名格式：`维度分析报告_YYYYMMDD_HHMMSS.xlsx`）

### 2. 前置条件

确保 `screenshots` 目录下有以下格式的 JSON 文件：
- `剧名_2年.json` - 近2年的数据
- `剧名_30天.json` - 近30天的数据

这些文件由 `dataeye_scraper.py` 自动生成。

## 输出报告结构

生成的 Excel 报告包含 3 个工作表：

### Sheet 1: 维度汇总表

核心指标一览，包含：
- 剧名
- 2年累计曝光量
- 30天累计曝光量
- 2年素材总数
- 30天在投素材数
- 生命周期天数
- 活跃天数
- 投放国家数
- Top5 国家
- 主要制作方
- 首次/最后投放日期

### Sheet 2: 详细数据明细

完整的维度数据，包含：
- 所有曝光量和下载量数据
- 投放国家详细分布
- 投放平台分布（FB/TT/Google等）
- 素材形式占比（视频/图片）
- 2年和30天数据对比

### Sheet 3: 制作方分析

制作方维度分析，包含：
- 制作方名称
- 投放剧目数
- 剧目列表
- 决策价值评估
  - ⭐ S级制作方（ReelShort、DramaBox等头部玩家）
  - 🔥 高频投放（5部以上）
  - 📊 中等投放（3-4部）
  - 📌 小规模投放（1-2部）
